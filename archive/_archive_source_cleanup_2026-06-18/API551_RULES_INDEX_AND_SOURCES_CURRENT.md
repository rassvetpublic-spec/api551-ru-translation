# API 551 — индекс правил, политик и источников CURRENT

Дата версии: 2026-06-16

Назначение: читаемый индекс текущей системы правил API 551. Этот пакет является clean-вариантом v5: в нём убраны промежуточные имена v3/v4/v5 из названий рабочих файлов, а PDF-правила оставлены в canonical approved-имени.

## 1. Файлы пакета

- `API551_RULES_01_FINAL_HTML_CURRENT.md` — финальный HTML.
- `RULES_API551_PDF_FORMATTING_APPROVED.md` — финальный PDF, approved-правила.
- `API551_RULES_03_WORKING_HTML_QA_CURRENT.md` — рабочие и проверочные HTML.
- `API551_RULES_04_FIGURE_COMPOSITE_OBJECTS_CURRENT.md` — рисунки как составные объекты.
- `API551_RULES_05_FIGURE_LABEL_REGISTER_CURRENT.md` — реестр надписей на рисунках.
- `API551_FIGURE_SPECIFIC_RULES_CURRENT.md` — живой изменяемый файл правил по конкретным рисункам.
- `CHECKLIST_API551_PDF_FORMATTING_APPROVED.md` — approved-чек-лист PDF.
- `CHECKLIST_API551_PDF_FORMATTING_APPROVED.json` — машинный approved-чек-лист PDF.

## 2. Ключевые решения

- Финальный HTML — русский чистовик с раскрываемым оригиналом по клику на абзац или объект.
- Клик по ссылке не раскрывает оригинал; ссылка работает как ссылка.
- В финальном HTML под рисунком показывается только approved-рисунок с переведёнными надписями.
- Все рисунки являются составными объектами.
- Минимальный состав рисунка: approved-рисунок + название/подпись снизу.
- Финальный PDF — только русский чистовик, в основном для печати.
- Figure-specific rules хранятся рядом с правилами, но помечены как живой изменяемый файл.
- Реестр надписей описывает и текстовые правила, и обязательную структуру CSV/JSON-полей.
- `VERIFIED_v2` PDF rules/checklist заменяются approved-файлами из этого пакета после принятия пользователем.

## 3. Что учтено из VERIFIED_v2

В `RULES_API551_PDF_FORMATTING_APPROVED.md` перенесены и усилены правила:

- главный запрет на потерю информации;
- назначение финального PDF;
- потоковая русская вёрстка;
- формат страницы, поля, колонтитулы и нумерация;
- шрифты и размеры;
- абзацы, заголовки и списки;
- технические переносы;
- таблицы и перенос таблиц;
- рисунки/Figure;
- размещение объектов;
- стоп-элементы при заполнении пустот;
- раздел-владелец объекта;
- NOTE и сноски;
- формулы, размеры и единицы;
- ссылки и PDF-аннотации;
- содержание, перечни рисунков и таблиц;
- запрещённое служебное содержимое;
- проверка после генерации;
- критические условия браковки;
- правило выдачи пользователю.

## 5. Что можно удалить после принятия clean-пакета

Можно удалить старые промежуточные правила:

- `RULES_API551_PDF_FORMATTING_VERIFIED_v2.md`;
- `CHECKLIST_API551_PDF_FORMATTING_VERIFIED_v2.md`;
- `REPORT_API551_PDF_FORMATTING_VERIFICATION_v2.md`;
- v3/v4/v5 промежуточные ZIP-пакеты правил, если этот clean-пакет загружен и принят;
- старые memory-audit файлы и v3-своды, если они полностью заменены этим пакетом.

Нельзя удалять исходники, master CSV, Stage2/Stage3 source packages и approved-registers.
