# API 551 — живой файл правил для конкретных рисунков v2

Дата версии: 2026-06-16

Статус: живой изменяемый файл. Этот файл хранится рядом с общей системой правил, но не считается стабильным архивным сводом. Он может часто меняться в ходе текущей приёмки рисунков.

## 1. Общий статус

- Рисунки приняты до Figure 27 включительно.
- Для Figure 27 исправлено NOTE 2.
- Следующие приоритетные проходы: Figure 29, Figure 32, Figure 33, Figure 38, Figure 39, Figure 44, Figure 47.
- После принятия всех рисунков нужно сформировать финальный approved-пакет/реестр рисунков.
- Финальный approved-пакет/реестр рисунков является обязательным источником для окончательной сборки переведённого документа.

## 2. Правила файла

- Этот файл фиксирует только правила для конкретных Figures.
- Общие правила находятся в v4-файлах.
- Если правило конкретного Figure конфликтует с общим правилом, конфликт выносится на ручное утверждение.
- Если правило конкретного Figure конфликтует с approved-register, приоритет имеет последний явно утверждённый approved-register.
- Старые чаты и память не являются источником истины без переноса в этот файл или approved-register.

## 3. Figure 1

- Строки вида `B bore diameter ... P external threaded connection` не внедрять в графический объект без утверждения.
- Такие строки допускается выносить в реконструированную таблицу из двух колонок.
- Статус: требует сверки с текущим approved-register.

## 18. Что нужно перенести из approved-register

- полный список Figures;
- статус каждого Figure;
- crop bbox;
- asset path/id;
- caption original;
- caption RU;
- labels list;
- notes list;
- table reconstruction rules;
- section/page mapping;
- first mention locations;
- protected zones;
- do-not-change zones.

После появления финального approved-register этот файл используется как ручной комментарий к реестру, а не как замена реестра.
