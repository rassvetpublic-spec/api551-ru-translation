# Техническое задание

**Проект:** перевод технической документации API 551 на русский язык  
**Документ-источник:** `API 551 2016 (R2024).pdf`  
**Рабочее название:** `API 551 — Process Measurement — RU Translation Project`  
**Статус ТЗ:** APPROVED-DRAFT для переноса в новый чат и дальнейшей работы  
**Дата фиксации:** 2026-06-01

---

## 1. Цель проекта

Создать русскую печатную версию английского PDF-документа API 551 с технически корректным переводом, сохранением структуры документа, внутренних ссылок, таблиц, рисунков и примечаний.

Итоговая цель проекта:

1. Получить полный русский HTML/JSON-документ для проверки.
2. Затем получить печатную PDF-версию на русском языке.
3. На финальном этапе заменить оригинальные графические объекты на реконструированные русские графические объекты, если они будут утверждены отдельно.

---

## 2. Главный принцип обработки

Основной текст документа переводится не по встроенному текстовому слою PDF, а по оптическому анализу страниц.

Рабочий процесс:

```text
Рендер страницы PDF → OCR → разметка блоков → корректировка OCR → перевод → HTML/JSON → проверка → финальная сборка
```

Запрещено использовать встроенный текстовый слой PDF как основной источник разметки и перевода текста. Его можно использовать только как вспомогательный источник для проверки спорных мест, но не как первичный источник.

## 4. Этапы проекта

## Stage 1 — перевод основного текста и вставка оригинальных Figure

### 4.1. Цель Stage 1

Создать промежуточный русский HTML/JSON-документ, в котором:

1. основной текст переведён на русский язык;
2. страницы разбиты на пронумерованные OCR-блоки;
3. Figure вставлены как оригинальные crop-изображения;
4. внутренние надписи Figure не заменяются;
5. внутренний анализ Figure сохранён в JSON;
6. все внутренние ссылки сохранены и работают;
7. обычные таблицы документа реконструированы как HTML-таблицы.

### 4.2. Что переводится на Stage 1

Переводить:

- основной текст документа;
- заголовки разделов и подразделов;
- списки;
- обычные таблицы документа, не встроенные в Figure;
- сноски;
- NOTE основного текста;
- подписи Figure текстом под оригинальным crop, если это нужно для читаемости русского HTML.

### 4.3. Что не переводится на Stage 1

Не переводить внутри Figure:

- надписи на чертежах;
- выноски;
- одиночные строки текста на графике;
- встроенные абзацы внутри Figure;
- встроенные таблицы внутри Figure;
- NOTE, входящие в crop Figure;
- табличные блоки, являющиеся частью Figure.

На Stage 1 Figure является неделимым вставляемым объектом.

---

## 5. Разметка страниц Stage 1

Каждая страница обрабатывается как изображение.

Минимальная структура страницы:

```text
Page N
└─ Text block pNNN_b001_text
```

Если на странице есть Figure:

```text
Page N
├─ Text block before Figure
├─ Figure placeholder
└─ Text block after Figure
```

Если на странице несколько Figure или таблиц:

```text
Page N
├─ Text block 001
├─ Figure placeholder
├─ Text block 002
├─ Table block
├─ Text block 003
└─ Figure placeholder
```

Порядок блоков должен соответствовать визуальному порядку сверху вниз.

---

## 6. Типы блоков страницы

Допустимые типы блоков в JSON:

| Тип | Назначение |
|---|---|
| `text` | основной текст, заголовки, списки |
| `table` | обычная таблица документа, не встроенная в Figure |
| `figure_placeholder` | место вставки оригинального Figure-crop |
| `footnote` | сноска страницы |
| `note` | примечание основного текста |
| `header` | верхний колонтитул, не переводится |
| `footer` | нижний колонтитул, не переводится |
| `page_number` | номер страницы, не переводится |
| `unclassified` | блок для ручной проверки |

## 7. Правила обработки Figure на Stage 1

1. Верхняя и нижняя граница каждого Figure берётся из утверждённого JSON-реестра Figure.
2. Оригинальный crop Figure вставляется в HTML как изображение.
3. Внутренние надписи Figure не заменяются.
4. Под Figure можно выводить русскую подпись текстом.
5. Внутренний анализ Figure сохраняется в JSON, но не выводится в основной Stage 1 HTML.
6. Если ссылка в основном тексте ведёт на Figure, ссылка должна вести на якорь соответствующего Figure в русском HTML.
7. Если целевой Figure находится вне текущего HTML-пакета, ссылка временно ведёт на страницу исходного PDF.

Пример JSON-записи Figure:

```json
{
  "figure_id": "fig020",
  "figure_no": 20,
  "title_original": "Figure 20—Classical Venturi Tube Dimensions",
  "title_ru": "Рисунок 20 — Размеры классической трубы Вентури",
  "pdf_page": 77,
  "api_page": 69,
  "crop_bbox_page_px": [0, 0, 0, 0],
  "original_crop_asset": "figures/fig020_original.png",
  "stage1_mode": "insert_original_as_image",
  "translate_inside_figure": false,
  "internal_analysis_available": true
}
```

## 8. Таблицы документа

Обычные таблицы документа, не входящие в Figure, на Stage 1 должны быть распознаны OCR-ом и реконструированы как HTML-таблицы.

Правила:

1. сохранять структуру строк и столбцов;
2. сохранять объединённые ячейки `rowspan` / `colspan`, если они определены;
3. шапки таблиц выравнивать по центру;
4. включать автоперенос текста в ячейках;
5. использовать минимальные вертикальные и горизонтальные отступы;
6. числовые значения и чистые размерные значения не переводить;
7. `in.` / `inch` в шапках таблиц переводить как `дюймы`;
8. в теле таблиц не заменять автоматически символ дюйма на слово `дюймы`;
9. таблица не должна выходить за правую границу страницы/контейнера;
10. если таблица физически не помещается, использовать внутреннюю горизонтальную прокрутку в HTML.

---

## 9. NOTE и примечания

Правила:

1. Полное примечание переводить как `ПРИМЕЧАНИЕ N.`.
2. Ссылку на примечание в тексте, таблице или выноске сокращать как `Прим. N`.
3. `NOTE` внутри Figure и `NOTE` основного текста являются разными целевыми объектами.
4. Группы `NOTE 1`, `NOTE 2`, `NOTE 3` не объединять в одну строку.
5. Каждое примечание должно быть отдельным блоком.
6. Если блок `NOTE` относится к Figure, на Stage 1 он остаётся внутри оригинального crop, но его анализ сохраняется в JSON.

---

## 10. Внутренние ссылки

Перевод не должен разрушать ссылочную структуру английского документа.

Нужно распознавать и сохранять ссылки на:

- Figure / Fig. / Figures;
- Table / Tables;
- Section / Clause / номера разделов;
- NOTE / Note;
- страницы исходного PDF, если ссылка используется как контроль источника.

Каждая найденная ссылка в JSON должна иметь статус:

| Статус | Значение |
|---|---|
| `resolved` | цель найдена в текущем русском HTML |
| `resolved_outside_pilot` | цель вне текущего HTML-пакета, но известна по registry |
| `external_reference` | ссылка на внешний стандарт/API/ASME/IEC/ISO |
| `ambiguous` | цель неоднозначна |
| `unresolved` | цель не найдена |
| `not_linked_by_rule` | ссылка намеренно не преобразована в гиперссылку |

Формат якорей:

```text
#fig-020
#table-001
#sec-4-2-3
#note-p033-002
```

Если цель есть в текущем HTML:

```html
<a href="#fig-020">рисунок 20</a>
```

Если цель вне текущего HTML-пакета:

```html
<a href="API%20551%202016%20(R2024).pdf#page=77">рисунок 20</a>
```

## 13. HTML Stage 1

Промежуточный HTML должен содержать:

1. навигацию по страницам и Figure;
2. блоки оригинального OCR-текста;
3. блоки русского перевода;
4. Figure placeholders с оригинальными изображениями;
5. рабочие внутренние ссылки;
6. отчёт по найденным ссылкам;
7. регулируемые параметры просмотра:
   - размер шрифта 9–16 pt;
   - вертикальный отступ ячеек;
   - горизонтальный отступ ячеек;
   - максимальная ширина изображений;
8. режим показа/скрытия raw OCR;
9. поиск по документу.

Для переносимости в новый чат предпочтителен самодостаточный HTML с изображениями, встроенными как `base64`.

## 14. Stage 2 — перевод и реконструкция Figure

Stage 2 выполняется после утверждения Stage 1.

Цель Stage 2 — подготовить второй реестр переведённых графических объектов.

Внутри Figure обрабатываются:

| Элемент внутри Figure | Режим Stage 2 |
|---|---|
| подпись Figure | переводить |
| встроенные абзацы | переводить текстом |
| встроенные таблицы | реконструировать HTML-таблицей |
| таблица со встроенным рисунком | реконструировать таблицу, рисунок оставить в нужной ячейке |
| NOTE-блоки | делать отдельной таблицей NOTE |
| одиночные надписи на чертеже | переводить после отдельного решения |
| чистые размеры, символы, числа | не переводить, хранить как служебные элементы |

Важно: одиночные надписи на чертеже на Stage 1 не трогать. На Stage 2 они переводятся только после отдельного решения о методе: таблица расшифровки под рисунком или наложение перевода поверх оригинала.

## 15. Реконструкция Figure на Stage 2

Для Stage 2 использовать справочные наработки:

- `API_551_image_register_v40.json`
- `API_551_image_register_RU_v40.html`
- `API_551_figures_full_reconstructed_register_v42.json`
- `API_551_figures_full_reconstructed_register_RU_v42.html`
- `RULES_Figure.md`

Правила Stage 2:

1. Исходный чертёж не искажать.
2. Для первой версии Stage 2 допускается оставлять оригинальный чертёж как изображение, а ниже давать расшифровку надписей.
3. Встроенные таблицы реконструировать как таблицы, а не как плоский список строк.
4. Если в таблице есть вложенный рисунок, реконструировать таблицу и вставлять оригинальный вложенный рисунок в нужную ячейку.
5. Многострочные пояснения переводить как единый смысловой блок.
6. NOTE внутри Figure оформлять отдельной таблицей.
7. Чистые размеры, числовые шкалы и одиночные символы не выводить в таблицу перевода, но хранить в JSON.
8. Для ссылок на примечания использовать `Прим. N`; для полного NOTE использовать `ПРИМЕЧАНИЕ N.`.

## 16. Печатный PDF

PDF не собирать до утверждения HTML/JSON-карты.

Финальный PDF выполняется после:

1. утверждения OCR-карты документа;
2. проверки ссылок;
3. проверки таблиц;
4. проверки Figure placeholders;
5. подготовки второго реестра переведённых Figure;
6. утверждения правил вёрстки.

Параметры печатной версии:

- основной шрифт: 12 pt;
- заголовки: 14 pt;
- таблицы: допускается 11 pt, если требуется уплотнение;
- минимальные отступы;
- автоперенос текста в таблицах;
- рабочие внутренние ссылки;
- исходные Figure на Stage 1 не изменять.
